The goal is to have a piece of machine code execute from within the language you are programming in. This was done all the time on the Commodore 64 to provide much faster processing within BASIC programs. This example simply has the machine code add two numbers and place the result in a memory location which is later read and displayed to the screen.

Alternate Solution

10 print chr$(147);
15 ml=49152
20 if peek(ml)<>24 and peek(ml+4)<>96 then gosub 100
30 poke 780,12:rem a register argument
35 poke 781,7:rem x register argument
40 sys 49152
45 a=peek(780):print a
50 end
100 for ad=49152 to 49156
110 read b
120 poke ad,b
130 next
140 return
9152 data 24:       rem clc
9153 data 109,13,3: rem adc $030d
9156 data 96:       rem rts

After some research (possibly when watching one of Robin Harbon's videos) I realized that Commodore BASIC has dedicated memory locations that allow the programmer to preset CPU register values for when the SYS command is executed. When the machine code returns control to BASIC, the registers are then unloaded into these same memory values, making a convenient pipleline for passing parameters.

In this example, whatever value is in location 780 will be loaded into the .A register (acumulator), and the value in location in 781 is loaded into the .X register. With the registers pre-loaded at this point, the machine code only needs to clear the carry flag and perform the "Add with Carry" (ADC) operation, which puts the result in the .A register.

On exit, the value is then transferred back to location 780 for reading by BASIC.
