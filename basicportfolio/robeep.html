I had learned how to manipulate sound out of the Commodore 64's SID chip. The program loops through random characters until it is able to spell a certain word.

The screen memory codes for each letter is not the same as the ASCII codes used to build strings. Characters start at 0 for the @ sign, then 1 for A, 2 for B, etc. I'm not really sure why I chose to do it this way, but if obfuscating the word was the goal, then the program was a success.
